Basic Instructions

1Pre-requisites
  • Run the sample applications on a Linux 64 bit Operating system (or Mac OSX 64 bits).
  • You will need to have Java OpenJDK (or Sun JDK) installed.
2Using Terminal
  • Each sample app includes instructions for running commands in terminal mode. Simply open a terminal window, copy and paste the provided command string and hit <enter>
  • The provided command will take care of running your sample. However, note that the samples will kill any running VoltDB database - do not run the samples on a production server.
3Questions?
  • Check out the VoltDB community forums for answers.
  • You can also find detailed documentation and white papers on VoltDB here.

Instructions

1Running the sample

Open Terminal, and run:
launch voter
This will open two Terminal windows:
  • A VoltDB database process.
  • A simulated call center client capturing phone calls.
2Monitor Performance

Next, click here to launch the real-time dashboard and see who's winning the vote.

Or, click here to launch VoltDB Studio.

The performance monitor will show you how fast VoltDB is processing phone calls.

3What is happening?
  • The call center is randomly posting real-time votes.
  • VoltDB validates each vote, blocking fraud.

The charts in VoltDB Studio (right panel) show how many votes the database is processing (transactions/second) and how long it takes to process each vote (latency in milliseconds).

The obvious goal: high transaction throughput with low latency (more votes, faster!)

At the end of the benchmark, the call center simulator reports the contest winner (in the Terminal window), a query that takes a mere fraction of a second, despite millions of votes having been processed!

4Explore...

You can open a New Query in VoltDB Studio to run requests against the database even as the Voter application is running. Click the Execute Query button after typing in your query.

Follow live voting results:
exec Results
Get the total number of votes:
select count(*) from votes;

To close VoltDB Studio, click here

5Get hands-on with Voter

You can customize the Voter sample by editing:

  • Change the number of contestants
  • Change the voting duration

Instructions

1Running the sample

Open Terminal, and run:
launch voltkv
This will open two Terminal windows:
  • A VoltDB database process.
  • A client benchmark application.
2Monitor Performance

Next, click here to launch VoltDB Studio.

The performance monitor will show you how fast VoltDB is processing GET/PUT calls.

3What is happening?
  • The client runs various PUT/GET operations (12K payload).
  • VoltDB serves as a simple high-speed data store.

The charts in VoltDB Studio (right side) show how many operations (transactions/second) the database is processing and how long it takes to process each operation (latency in milliseconds).

The obvious goal: high transaction throughput with low latency (more operations, faster!)

At the end of the benchmark, the client application reports performance results (in the Terminal window)

4Explore...

You can open a New Query in VoltDB Studio to run requests against the database even as the application is running. Click the Execute Query button after typing in your query.

Sample stored data:
select top 10 * from store;
Get the total number of stored values:
select count(*) from store;

To close VoltDB Studio, click here

You can customize the key-value store sample by editing:

  • Test with GZIP compression.
  • Test with various PUT/GET ratios.
