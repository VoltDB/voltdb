2Run the voter client

This will run a benchmark application that connects to the database and generates transactions that simulate a call center capturing text message votes.
3Monitor Performance

Next, click here to launch the real-time dashboard and see who's winning the vote.

Or, click here to launch VoltDB Studio. Then click on the toolbar button to view real-time performance statistics.

4What is happening?
  • The call center is randomly posting real-time votes.
  • VoltDB validates each vote, blocking fraud.

The charts in VoltDB Studio (right panel) show how many votes the database is processing (transactions/second) and how long it takes to process each vote (latency in milliseconds).

The obvious goal: high transaction throughput with low latency (more votes, faster!)

At the end of the benchmark, the call center simulator reports the contest winner (in the Terminal window), a query that takes a mere fraction of a second, despite millions of votes having been processed!

5Explore...

You can click on the New Query toolbar button in VoltDB Studio to open a new query window and run requests against the database as the Voter application is running. Click the Execute Query button after typing in your query.

You can customize the Voter sample by editing:

Find the function client section and edit the parameters to:

  • Change the number of contestants
  • Change the voting duration
  • To observe lower latency, set autotune=false and ratelimit to 90-95% of the throughput observed with autotune=true.

4What is happening?
  • The client runs various PUT/GET operations (12K payload).
  • VoltDB serves as a simple high-speed data store.

The charts in VoltDB Studio (right side) show how many operations (transactions/second) the database is processing and how long it takes to process each operation (latency in milliseconds).

The obvious goal: high transaction throughput with low latency (more operations, faster!)
