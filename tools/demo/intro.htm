Vote with the Stars!

Imagine a prime-time TV talent show.
  • Six artists
  • Millions of viewers
  • Two minutes to vote for your favorite contestant
The application must scale from zero to millions of simultaneous callers, tracking all their votes while protecting against fraud.
  • Each caller is allowed a maximum of three votes
  • Callers can only vote for the same contestant once
  • The system must report a winner in less than one minute
See the instructions in the left panel to explore the Voter sample.

Key/Value Store

VoltDB is a fully featured RDBMS. However, if you need a high performance Key/Value store backed by a highly-available, ACID compliant database manager, this example will show you how to get the best of both worlds by using:
  • String keys
  • Binary values up to 1MB
Binary storage lets you use whatever logic you may need on the client side. For instance, you can:
  • Serialize client objects in binary form
  • Store zipped content/files
  • Store Unicode string data
See the instructions in the left panel to explore the K/V Store sample.
